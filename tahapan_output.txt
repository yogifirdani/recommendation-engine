representasi kata, sehingga kata seperti “Wisata”, “WISATA”, atau “wisata”

Tabel 4.4  Proses Case Folding  
def case_folding(text):  
    if not text:  
        return ""  
 
    text = text.lower()  
    text = re.sub(r'[^a -zA-Z0-9\s]', ' ', text)  
    text = re.sub(r' \s+', ' ', text)

Pada Tabel 4.4 merupakan kode dari proses Case Folding  yang digunakan untuk

 4.1.3 Hasil TF -IDF 
Setelah semua data deskripsi paket wisata melalui tahapan pre-processing  (yang 
mencakup case folding , stopword removal , stemming menggunakan algoritma PySastrawi , 
dan tokenizing ), diperoleh kumpulan kata unik (vocabulary) yang membentuk representasi 
fitur dari koleksi dokumen database. Kata -kata yang telah dibersihkan ini digunakan 
sebagai dasar dalam membangun representasi numerik dokumen menggunakan teknik 
pembobotan TF -IDF ( Term Frequency - Inverse Document Frequency ). Pembobotan TF -
IDF bertujuan untuk menghitung seberapa penting atau relevan sebuah kata terhadap 
keseluruhan koleksi dokumen paket wisata. Sistem pada penelitian ini 
mengimplementasikan library Scikit -Learn  (TfidfVectorizer) yang secara otomatis 
memproses nilai TF dan IDF secara serentak.  
Menghitung seberapa sering sebuah kata (token) muncul di dalam satu dokumen paket 
wisata. Semakin sering kata tersebut disebut (akibat dari teknik Feature Engineering 
pengulangan kalimat deskripsi), maka bobot kemunculannya semakin inggi.  
Tabel 4.1 2 Proses TF-IDF 
vectorizer = TfidfVectorizer(  
    analyzer='word',

    min_df=1,  
    sublinear_tf=True

Pada Tabel 4.10 menjalankan algoritma TF -IDF. Parameter ngram_range=(1, 2) diatur

 4.1.4 Hasil Cosine Similarity  
Setelah proses pembobotan TF -IDF menghasilkan representasi matriks numerik untuk 
setiap paket wisata di database, langkah selanjutnya adalah mengukur tingkat kemiripan 
antara preferensi yang diketikkan pengguna dengan seluruh paket wisata yang tersedia. 
Pengukuran ini dilakukan menggunakan metode Cosine Similarity . Cosine Similarity  
bekerja dengan cara merepresentasikan teks input pengguna dan teks deskripsi paket wisata 
sebagai dua buah vektor dalam ruang multidimensi. Algoritma kemudian menghitung nilai 
cosine dari sudut (angle) yang terbentuk di antara kedua vektor tersebut. Hasil  dari 
perhitungan ini digunakan sebagai dasar pijakan mutlak (core engine) bagi sistem dalam 
menentukan peringkat (ranking) rekomendasi, di mana paket wisata yang memiliki vektor 
dengan arah paling selaras (sudut terkecil) dengan niat pengguna akan ditampi lkan pada 
urutan teratas.  
Table 4.1 3 Proses Cosine Similarity  
def calculate_similarity(preference_vector, package_vectors_list):  
  
        pkg_matrix = np.array(package_vectors_list)  
        pref_vector_2d = preference_vector.reshape(1, -1) 
        similarity_matrix = cosine_similarity(pref_vector_2d, pkg_matrix)  
        scores = similarity_matrix[0].tolist()  
 
    return scores  
Berbeda dengan beberapa pendekatan konvensional yang menghitung kemiripan 
menggunakan rumus matematis manual baris -per-baris, sistem pada penelitian ini 
dirancang untuk menangani komputasi tingkat lanjut dengan memanfaatkan pustaka scikit -
learn  (sklearn.metrics.pairwise.cosine_similarity). Fungsi pada Tabel 4.1 3  menerima 
matriks vektor preferensi pengguna (pref_vector_2d) dan membandingkannya sekaligus 
dengan seluruh kumpulan matriks vektor paket wisata dari database (pkg_matrix). Hasil 
akhir dari fungsi ini berupa nilai cosine similarity  yang berada dalam rentang skor 0 hingga 
1. Nilai yang semakin mendekati 1 menunjukkan tingkat kemiripan yang sangat tinggi 
(vektor saling berhimpit), sedangkan nilai yang mendekati 0 menunjukkan tidak adanya 
relevansi kata sama sekali (vektor tegak lurus).
